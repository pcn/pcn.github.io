5/6/2015

Today I've been playing with creating a somewhat workable docker/container environment. I've been reading a whole lot of material about how to setup docker.
I'm doing this to support a small test system that I want to grow.

Phase 1: Install containers for the head version of antirez/disque.
Doing this gives me a service that I can easily read/write to, which is simple, but which doesn't have a whole lot of convetion build up around it from other contributors. This lets me make all of my own mistakes.
The first part of the project is to create a 3-node consul service. Once that service is established, my hope is to build other experiments around it.

To allow for discovery, each VM that's running disque will run consul, regstrator, and I don't know, something else.
Other things to run here:

  • Salt deployments of services
  • Use runit to start and run containers that provide services.
  • Template runit services
  • Use consul/registrators registrations within to configure services
It seems like there are a number of different approaches to starting services. I'm most comfortable starting e.g. registrator under runit.
E.g. /etc/sv/registrator/run: 
        #!/bin/bash
        docker run -d=false -t=true -v /var/run/docker.sock:/tmp/docker/sock -h $HOSTNAME gliderlabs/registrator consul:
